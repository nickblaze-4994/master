AI Workflow Orchestration Projects

A single page for my AI workflow prototypes deployed on Vercel. These projects explore support automation, MCP-style agent workflows, vendor operations, guided intake, operational routing, and AI-assisted decision flows.

Background

These projects focus on using AI as a workflow layer between messy business inputs and structured operational actions. Each prototype takes an input, applies AI or deterministic workflow logic, and turns it into a clearer next step for a user or team.

Latest Project

Support Agent MCP

A support workflow prototype built around an MCP-style agent flow. The project demonstrates how a support agent can receive a customer issue, reason through the request, call the right backend logic, and return a grounded response.

The latest deployment is connected to the GitHub repo and running as the current Vercel production deployment. It includes environment-backed API access and a live production domain.

Support Automation

Woot Support Agent

A customer support workflow prototype for Woot-style support scenarios. The agent handles user questions, checks the relevant policy path, and shows the decision process through an activity panel.

The project demonstrates an LLM tool-calling loop combined with deterministic policy checks, so the user can see both the answer and the workflow behind it.

Vendor Operations

Vendor Ops Agent

A vendor operations workflow prototype focused on routing, triage, and operational handoffs. It is designed around repetitive vendor-facing issues that usually require manual review.

The project shows how AI can sit between incoming requests and the next operational step, helping structure the work before a human reviews it.

Guided Intake

Guided Intake

A guided intake workflow that turns user inputs into structured classifications and routing decisions. It is built around the idea that intake forms can be more useful when AI helps interpret context instead of only collecting fields.

The project demonstrates how a queue of intake items can be processed, classified, and routed in a clear interface.
